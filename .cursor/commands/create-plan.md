# Plan Creation Stage

**目的**: 基于充分的探索，生成一个带有状态跟踪的 markdown 执行计划。

## 计划要求

- 包含清晰、最小、简洁的步骤
- 使用这些表情符号跟踪每个步骤的状态：
  - ✅ 完成
  - 🟡 进行中
  - ⬜ 待办
- 包含整体进度的动态跟踪（百分比）
- **不要**添加额外范围或不必要的复杂性
- 步骤应该是模块化、优雅、最小且可增量测试的

## Markdown 模板

```markdown
# 功能实现计划

**总体进度**: `0%`

## TLDR
[我们正在构建什么以及为什么的简短摘要]

## 实现步骤

### ⬜ 步骤 1: [步骤名称]
**目的**: [这一步实现什么]
**文件**:
- `path/to/file.ts`

**任务**:
- [ ] 子任务 1
- [ ] 子任务 2
- [ ] 子任务 3

---

### ⬜ 步骤 2: [步骤名称]
**目的**: [这一步实现什么]
**文件**:
- `path/to/file.ts`

**任务**:
- [ ] 子任务 1
- [ ] 子任务 2

---

### ⬜ 步骤 3: [测试 & 验证]
**目的**: 确保一切按预期工作
**任务**:
- [ ] 测试主要功能
- [ ] 测试边缘情况
- [ ] 验证性能
- [ ] 更新文档

---

## 风险 & 注意事项
- [风险 1 及缓解措施]
- [风险 2 及缓解措施]

## 成功标准
- [ ] 标准 1
- [ ] 标准 2
- [ ] 标准 3
```

## 重要提示

1. **保持最小化**: 不要过度工程化或添加不必要的步骤
2. **可测试**: 每个步骤都应该可以独立测试
3. **跟踪进度**: 随着实施更新状态表情符号和百分比
4. **清晰的目的**: 每个步骤都应该有明确的目的
5. **模块化**: 步骤应该是可以独立完成的逻辑单元

创建计划后：
- 将其保存到 `plans/[feature-name].md`
- 等待批准再开始执行

# Initial Exploration Stage

**目的**: 在编写任何代码之前理解问题。这是"先思考再构建"的检查点。

## 你的任务

你的任务**不是**立即实现功能，而是充分理解它。

## 你的职责

- 分析并理解现有代码库
- 准确确定这个功能如何集成，包括所有依赖项
- 清楚地识别任何不清楚或模糊的内容
- 列出你需要回答的所有问题或歧义

记住，你的工作**不是**实现（还不是）。只是理解。

请确认你完全理解，我会开始下一阶段。

## 探索检查清单

在继续之前，请回答：

1. **问题理解**
   - 我们要解决什么问题？
   - 为什么需要这个功能？
   - 它如何适应更大的系统？

2. **技术范围**
   - 涉及哪些文件/模块？
   - 有哪些依赖关系？
   - 可能的副作用或影响？

3. **不明确之处**
   - 有什么不清楚的地方？
   - 需要澄清哪些假设？
   - 有哪些边缘情况需要考虑？

4. **现有模式**
   - 代码库中是否已有类似实现？
   - 应该遵循哪些现有模式？
   - 需要考虑哪些架构约束？

## 输出

提供一个简洁的探索摘要：

```markdown
## 探索摘要

### 问题理解
[对问题和目标的清晰陈述]

### 技术范围
- 受影响的文件: [列表]
- 关键依赖: [列表]
- 潜在影响: [列表]

### 需要澄清
1. [问题 1]
2. [问题 2]
3. [问题 3]

### 现有模式
[相关的现有实现或模式]

### 建议的方法
[实现的高层次方法]
```
